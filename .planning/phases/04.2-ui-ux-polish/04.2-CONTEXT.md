### Detailed Root Cause: Tree → Inspector Flow

**The Problem:**

1. `MemoryTree` component builds a tree from `/keys` endpoint, which returns `KeyNode` objects
2. Each `KeyNode` has `id` property that is the **key path** (e.g., `/test/batch-1`)
3. When user clicks a memory leaf node, `memory-tree.tsx` calls:
   ```typescript
   setSelectedMemory(node.id) // node.id = "/test/batch-1"
   setInspectorOpen(true)
   ```
4. `InspectorPanel` receives `selectedMemory = "/test/batch-1"` (a key path)
5. Inspector uses `useMemory()` hook which calls `/memories/:id` endpoint
6. Backend expects UUID in `:id` param, gets `/test/batch-1`, returns 404

**The Fix:**

1. Add `/memories/key/:key` endpoint to backend (already done in Plan 04-02)
2. Create `useMemoryByKey()` hook that calls the key-based endpoint
3. Update Inspector to detect UUID vs key path and use appropriate hook
4. Update Tree to pass memory UUID when available (key nodes don't have UUID)

**Current State:**
- `useMemoryByKey()` hook already exists in `use-memories.ts`
- Inspector already has logic to detect key paths and use appropriate hook
- The Tree still passes `node.id` (key path) instead of memory UUID

**Remaining Work:**
- Backend needs to return memory UUID in key tree response (or lookup)
- OR Tree needs to fetch memory details by key before opening inspector

## Implementation Decisions

### Decision 1: Inspector Opening Strategy
**ID:** D-04.2-01  
**Status:** Locked  
**Decision:** Use dual-mode inspector that accepts both UUIDs and key paths  
**Rationale:** Key tree nodes don't have UUIDs until fetched. Supporting both allows immediate opening without extra fetch.  
**Implementation:** Inspector checks `isKeyPath = selectedMemory?.startsWith('/')` and uses `useMemoryByKey()` for key paths, `useMemory()` for UUIDs.

### Decision 2: Pagination Strategy  
**ID:** D-04.2-02  
**Status:** Locked  
**Decision:** Implement cursor-based pagination with "Load More" button  
**Rationale:** UUID-based pagination is more reliable than offset for concurrent modifications. "Load More" is simpler than numbered pages for this use case.  
**Implementation:** Add `cursor` and `hasMore` to API response. UI shows cursor-based "Load More" button in table footer.

### Decision 3: Error Boundary Strategy
**ID:** D-04.2-03  
**Status:** Locked  
**Decision:** Wrap each route in ErrorBoundary, use ErrorCard for API errors  
**Rationale:** Route-level boundaries catch render errors. ErrorCard handles fetch errors with retry.  
**Implementation:** ErrorBoundary in route files, ErrorCard in data-fetching components.

### Decision 4: Keyboard Shortcuts
**ID:** D-04.2-04  
**Status:** Locked  
**Decision:** Global shortcuts via `useKeyboard` hook, scope-aware  
**Rationale:** Centralized shortcut management prevents conflicts. Scope awareness allows same key in different contexts.  
**Implementation:** `useKeyboard` hook with scope registration. Shortcuts: `?` (help), `/` (search focus), `esc` (close inspector), `r` (refresh), `n` (new memory).

### Decision 5: URL State Sync
**ID:** D-04.2-05  
**Status:** Locked  
**Decision:** Sync search query and selected memory to URL params  
**Rationale:** Enables shareable filtered views and browser back/forward navigation.  
**Implementation:** `useSearchParams` from React Router. Update URL on search change. Restore from URL on mount.

### Decision 6: Bulk Actions
**ID:** D-04.2-06  
**Status:** Locked  
**Decision:** Implement row selection in TanStack Table, enable bulk operations  
**Rationale:** TanStack Table has built-in row selection. Bulk operations need batch API support.  
**Implementation:** Enable `enableRowSelection` in table. Add toolbar with bulk Forget/Export buttons. Create `useBulkForget` mutation.

### Decision 7: Mobile Touch Targets
**ID:** D-04.2-07  
**Status:** Locked  
**Decision:** Minimum 44px touch targets, use `min-h-[44px]` and `min-w-[44px]`  
**Rationale:** WCAG 2.1 AA requires 44x44px minimum. Current buttons are often 32-36px.  
**Implementation:** Audit all interactive elements. Add min-dimensions to button classes. Increase spacing.

### Decision 8: Offline Detection
**ID:** D-04.2-08  
**Status:** Locked  
**Decision:** Use `navigator.onLine` API with online/offline events  
**Rationale:** Simple, well-supported API. Shows indicator when connection lost.  
**Implementation:** `useOnlineStatus` hook. Store in UI store. Show banner when offline.

## Technical Constraints

1. **TanStack Table v8**: Must use built-in sorting, selection, pagination features
2. **TanStack Query v5**: Mutations must invalidate appropriate query keys
3. **Zustand**: Persist only essential state (not selection, not loading states)
4. **Tailwind v4**: Use CSS-first configuration, glassmorphism utilities
5. **React Router v7**: Use `useSearchParams` for URL sync
6. **TypeScript**: All new hooks/components must be fully typed

## Risk Assessment

| Risk | Impact | Mitigation |
|------|--------|------------|
| Backend API changes needed for pagination | High | Check if `/memories` already supports cursor. If not, add offset pagination first. |
| Keyboard shortcuts conflict with browser | Medium | Use `preventDefault()` and `stopPropagation()`. Avoid single-key shortcuts that browser uses. |
| URL state sync causes unnecessary re-renders | Medium | Debounce search updates. Only update URL on blur or Enter key. |
| Bulk forget requires backend batch endpoint | Medium | If batch endpoint doesn't exist, implement client-side sequential forget with progress. |
