## Key Features

### 1. Keyboard Shortcuts System (use-keyboard.ts)
- **Global shortcuts:** `?` (help), `Esc` (close), `r` (refresh), `/` (focus search)
- **Scope-aware:** Modal > Inspector > Table > Search > Global priority
- **Prevents conflicts:** Automatically prevents default for registered shortcuts

### 2. URL State Synchronization (use-url-state.ts)
- **Bidirectional sync:** URL → Store on mount, Store → URL on change
- **Debounced updates:** 300ms delay to prevent excessive history entries
- **Shareable links:** Search query and selected memory persist in URL

### 3. Bulk Actions (bulk-action-bar.tsx)
- **Row selection:** Checkbox column with select-all functionality
- **Actions:** Forget (with confirmation), Export JSON, Cancel
- **Progress tracking:** Shows "Forgetting 3 of 5..." during operation

### 4. Empty States (empty-state.tsx)
- **Variants:** empty, search, filter, error
- **Helpful copy:** Context-specific messaging with action buttons
- **Consistent styling:** Glassmorphism theme matching

### 5. Context Menus (context-menu.tsx)
- **Right-click:** Positioned at click coordinates, constrained to viewport
- **Menu items:** Open in Inspector, Copy Key Path, Copy JSON, Forget
- **Auto-close:** Click outside or press Escape

### 6. Offline Detection (use-online-status.ts)
- **navigator.onLine API:** Browser-native online/offline events
- **Banner indicator:** Fixed amber banner when connection lost
- **Retry button:** Manual refresh to check connection status

### 7. Mobile Touch Targets
- **44px minimum:** All interactive elements meet WCAG 2.1 AA requirements
- **Responsive:** Smaller on desktop (32-40px), larger on mobile (44-48px)
- **Classes:** `min-h-[44px] md:min-h-[32px]` pattern

### 8. Search Filters (header.tsx)
- **Domain filter:** Dropdown with all memory domains
- **Author filter:** Text input for author name
- **Date range:** Last 24h, 7d, 30d, or all time
- **Active chips:** Visual indicator of applied filters

## Architecture Decisions

### Decision 1: Inspector Dual-Mode (D-04.2-01)
- **Rationale:** Tree nodes have key paths (`/test/batch-1`), API expects UUIDs
- **Solution:** Inspector checks `selectedMemory?.startsWith('/')` and uses appropriate hook
- **Result:** Tree click opens inspector without 404 errors

### Decision 2: Cursor-Based Pagination (D-04.2-02)
- **Rationale:** Offset pagination unreliable with concurrent modifications
- **Solution:** Use `hasMore` and `nextOffset` from API response
- **Result:** "Load More" button loads additional 50 items at a time

### Decision 3: Scope-Aware Keyboard Shortcuts (D-04.2-04)
- **Rationale:** Same key (e.g., `Esc`) should behave differently based on context
- **Solution:** Priority-based scope system: modal > inspector > table > search > global
- **Result:** Natural keyboard navigation without conflicts

### Decision 4: Mobile-First Touch Targets (D-04.2-07)
- **Rationale:** WCAG 2.1 AA requires 44x44px minimum for touch targets
- **Solution:** Responsive sizing: `p-2 md:p-0.5`, `min-h-[44px] md:min-h-[32px]`
- **Result:** Large touch targets on mobile, compact on desktop

## Implementation Details

### Bulk Actions Flow
```
User clicks checkbox → Row selected → BulkActionBar appears
User clicks Forget → Confirm dialog → Sequential forget API calls
Progress shown → Complete → Clear selection
```

### Context Menu Flow
```
User right-clicks row → preventDefault() → Calculate position
Render menu at (x,y) → User selects item → Execute action → Close menu
Click outside → Close menu
```

### Offline Detection Flow
```
App loads → Check navigator.onLine → Show/hide banner
'online' event → Hide banner, retry queued mutations
'offline' event → Show banner, queue mutations locally
```
