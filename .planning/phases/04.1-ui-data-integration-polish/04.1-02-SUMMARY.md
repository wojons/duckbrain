# Phase 04.1 Plan 02: Tree and Table Click Handler Fixes Summary

## Overview
Fixed Tree expand/collapse and Memory → Inspector click handlers. Added proper loading skeletons and error boundaries with retry functionality.

## Changes Made

### 1. Skeleton Components (NEW)
**File:** `packages/ui/src/components/ui/skeleton.tsx`

Created glassmorphism-styled loading skeletons:
- `Skeleton` - Base skeleton with optional shimmer animation
- `SkeletonCard` - Card-shaped skeleton for card content
- `SkeletonTable` - Table rows skeleton with configurable rows/columns
- `SkeletonTree` - Tree view skeleton with hierarchical structure
- `SkeletonList` - Simple list skeleton

All skeletons use `animate-pulse`, glassmorphism styling (`bg-white/5`), and optional shimmer effect.

### 2. Error Boundary Components (NEW)
**File:** `packages/ui/src/components/ui/error-boundary.tsx`

Created error handling components:
- `ErrorBoundary` - React class component error boundary
- `ErrorCard` - Reusable error card with retry button
- `AsyncErrorBoundary` - Specialized for async data fetching

ErrorCard features:
- Glassmorphism styling with red border (`var(--color-error)`)
- Alert icon with error color
- Error message display in monospace font
- Retry button with refresh icon
- Graceful fallback rendering

### 3. Memory Tree Fixes
**File:** `packages/ui/src/components/memory-tree.tsx`

**Loading State:**
- Replaced text with `SkeletonTree` component
- Shows 3 levels of skeleton folders

**Error State:**
- Replaced error text with `ErrorCard` component
- Added `refetch()` from useKeys hook for retry

**Click Handler Fixes:**
- Separated event handlers:
  - `handleToggleFolder` - Called when chevron clicked (stops propagation)
  - Folder content click - Toggles folder (stops propagation)
  - Memory click - Opens inspector (doesn't stop propagation for row)
- Each handler calls `e.stopPropagation()` appropriately
- Added comments explaining each handler's purpose

**Selected State:**
- Pass `selectedMemory` prop through all TreeNodeComponent levels
- Check if `selectedMemory === node.id` for leaf nodes
- Apply visual highlight: `bg-white/10 border border-azure/30`
- Use CSS custom properties for consistent theming

### 4. Memory Table Fixes
**File:** `packages/ui/src/components/memory-table.tsx`

**Loading State:**
- Replaced spinner with `SkeletonTable` component
- Shows 10 rows × 6 columns skeleton

**Error State:**
- Replaced error text with `ErrorCard` component
- Added `refetch()` from useMemories hook for retry

**Click Handler Fixes:**
- Changed `handleRowClick` signature to accept `React.MouseEvent`
- Added `e.stopPropagation()` and `e.preventDefault()`
- Verify `memory.id` exists before setting selected memory
- Prevent event bubbling to parent elements

**Selected State:**
- Get `selectedMemory` from UI store
- Check if `selectedMemory === memory.id` for each row
- Apply visual highlight with border-azure
- Add subtle background color (`rgba(0, 212, 255, 0.05)`) when selected

## Key Decisions

1. **Event Handler Separation**: Used separate handlers for chevron (toggle only), folder content (toggle), and memory (open inspector) to prevent propagation issues.

2. **Glassmorphism Consistency**: All skeletons and error cards use the existing glass-panel, glass-button styles with CSS custom properties.

3. **Selected State Propagation**: Tree passes selectedMemory through all levels since it's a recursive component. Table gets it directly from UI store.

4. **Retry Pattern**: Both components use TanStack Query's `refetch()` function for retry, which invalidates cache and re-fetches data.
